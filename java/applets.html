Applets

Applets are the graphical user interface, which are embedded in the webpage.

To embed applet in the webpage, we use an applet tag including the following attributes :-

  • Code : In code, we will spcify the path of the code file.
  • Width & Height : Because am applet is embedded in a webapge, we will spcify the width and height that will be taken in a webapge by an applet. for e.g 
    <applet code="A.class" width="400" height="400"></applet>

Applets are of two types :-

  1. Standalone Applet : These are the applets that are executed independently without any help of the browser. These are executed by using the appletviewer tool. For e.g 
    appletviewer A.java
  2. Embedded Applet : These are the applets that are executed with the help of the browser. For e.g 
    <applet code="A.class" width="400px" height="400px"></applet>

For creating an applet, we will use a class that is applet which present in a package called java.applet.Applet. Applet by default uses FlowLayout as a Layout Manager.

Applet Life Cycle

There is a life cycle of an applet which is consisting of five phases. Each phase is represented as a function Applet class.

  • Initialization : In this phase, we will initialize the variables that are required for an applet. If we want to initalize some variable in the applet, then we will use/override a function that is :- 
    public void init() {}
    This phase is executed only once.
  • Start : So when an applet is ready to run, then the start function of the applet is executed. To override this function, we wil use following method :- 
    public void start() {}
  • Paint : Applet is a graphical component, so each time the new pixels are painted. By increasing or decreasing the size of an applet, the paint is called multiple times. To override the paint function, we will use following syntax :- 
    public void paint (java.awt.Graphics) {}
  • Stop : When the applet is ideal that is when we minimise the applet, then stop function is called multiple times and when we restore the applet, the start function will be called. For overriding the stop function, we will write :- 
    public void stop() {}
  • Destroy : When we close an applet or we close the webpage which is containing applet, then destroy function is called. Destroy function is called only one time. 
    public void destroy() {}

Example

Here's a basic example of a Java applet that displays "Hello, Applet!" within a web page. 

import java.applet.Applet;
import java.awt.Graphics;

public class HelloApplet extends Applet {
    public void paint(Graphics g) {
        g.drawString("Hello, Applet!", 50, 25);
    }
}

Embedding Applets in HTML

To embed an applet in an HTML page, you can use the <applet> or <object> tag. Here's an example of embedding the "HelloApplet" in HTML. 

<html>
<head>
    <title>Applet Example</title>
</head>
<body>
    <applet code="HelloApplet.class" width="300" height="150">
    </applet>
</body>
</html>

Conclusion

Java applets provide a way to create interactive content within web pages. Understanding the applet life cycle and embedding applets in HTML is essential for web development with Java applets.
