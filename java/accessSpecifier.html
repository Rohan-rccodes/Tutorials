Access Specifiers in Java

Introduction to Access Specifiers

In Java, access specifiers (also known as access modifiers) are keywords used to control the visibility and accessibility of classes, methods, fields, and other members within a Java program. They define the scope and who can access certain elements of a class or interface. Java provides four main access specifiers:

  • Public: The most permissive access specifier. Members declared as public are accessible from any other class or package.
  • Private: The most restrictive access specifier. Members declared as private are accessible only within the same class.
  • Protected: Members declared as protected are accessible within the same class, the same package, and subclasses outside the package.
  • Default (no modifier): Also known as package-private. Members with no access specifier are accessible within the same class and the same package but not outside the package.

Using Public Access Specifier

When a member is declared as public, it can be accessed from anywhere in the program. Here's an example of a public method: 

public class MyClass {
    public void myPublicMethod() {
        // Code here
    }
}

Using Private Access Specifier

Members declared as private are only accessible within the same class. Here's an example of a private field: 

public class MyClass {
    private int myPrivateField;
}

Using Protected Access Specifier

Members declared as protected are accessible within the same class, the same package, and subclasses outside the package. Here's an example of a protected method: 

public class MyBaseClass {
    protected void myProtectedMethod() {
        // Code here
    }
}

Using Default (Package-Private) Access Specifier

Members with no access specifier (default) are accessible within the same class and the same package but not outside the package. Here's an example of a default (package-private) field: 

class MyPackageClass {
    int myPackageField; // Default access specifier
}

Conclusion

Access specifiers in Java provide control over the visibility and accessibility of class members. Choosing the appropriate access specifier is crucial for encapsulation, security, and maintaining the integrity of your Java programs.
