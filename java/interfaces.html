Interfaces

In Java, an interface is a blueprint for a class that defines a set of methods but does not provide the implementation of those methods. Interfaces allow you to achieve multiple inheritance and ensure that classes implementing the interface provide specific behaviors.

  • Interfaces are the collection of abstract functions and final static variables.
  • We cannot define any function in the interfaces.
  • If we don't write abstract keyword, then by default all the functions are abstract.
  • In interfaces, the members are by default public.
  • We cannot create an object of the interface.
  • An interface is inherited by a class or another interface.
  • When a class inherits an interface then it uses a keyword implements.

Defining an Interface

To define an interface in Java, you use the interface keyword followed by the interface name. Methods declared in an interface are implicitly public and abstract (no method body). 

interface Drawable {
    void draw(); // Abstract method
}

Implementing an Interface

To use an interface, a class must implement it using the implements keyword. The implementing class must provide concrete implementations for all the methods declared in the interface. 

class Circle implements Drawable {
    // Override
    public void draw() {
        System.out.println("Drawing a circle");
    }
}

Using Interfaces

You can create objects of classes that implement interfaces and invoke the methods defined in those interfaces. 

public class Main {
    public static void main(String[] args) {
        Drawable circle = new Circle();
        circle.draw(); // Calls the draw() method of Circle class
    }
}

Multiple Interface Implementation

A class can implement multiple interfaces in Java, allowing it to inherit and provide implementations for methods from multiple sources. 

interface Soundable {
    void makeSound();
}
class Dog implements Drawable, Soundable {
    // Overriding
    public void draw() {
        System.out.println("Drawing a dog");
    }
    public void makeSound() {
        System.out.println("Dog barks");
    }
}

Interface inheriting interface

When Interface inherits another interface, then it uses a keyword extends. An interface can inherit multiple interfaces.

interface A {
    void a();
}
interface B extends A {
    void b();
}

Class inheriting class and implementing interfaces

A class must inherit first then implement interfaces.

class C extends D implements A, B {
    // Implementing all the methods of A and B
    public void a() {
        System.out.println("a");
    }
    public void b() {
        System.out.println("b");
    }
}

Conclusion

Interfaces in Java are a powerful way to achieve abstraction and multiple inheritance. They define a contract that implementing classes must adhere to, ensuring that specific behaviors are provided. Interfaces play a key role in building flexible and extensible Java applications.
