Abstract Keyword

In Java, the abstract keyword is used to indicate that a class or method has a missing or incomplete implementation and cannot be instantiated directly. Abstract classes can contain abstract methods, which are declared without a body and must be implemented by concrete (non-abstract) subclasses. Class having a abstract function must have abstract keyword. Abstract class cannot be used to create objects directly. But we can make reference object of abstract class.

Using abstract with Classes

Abstract classes can have abstract methods as well as concrete methods. 

abstract class Shape {
    abstract void draw(); // Abstract method
    void resize() {
        System.out.println("Resizing the shape.");
    }
}

Using abstract with Methods

When applied to a method, the abstract keyword indicates that the method is an abstract method and must be implemented by any concrete subclass. Abstract methods are declared without a method body. 

abstract class Animal {
    abstract void makeSound(); // Abstract method
}
class Dog extends Animal {
    // Override
    void makeSound() {
        System.out.println("Dog barks.");
    }
}

Creating Subclasses of Abstract Classes

Subclasses of abstract classes must provide implementations for all abstract methods declared in the superclass. Once all abstract methods are implemented, the subclass becomes concrete and can be instantiated. 

public class Main {
    public static void main(String[] args) {
        Animal myAnimal = new Dog();
        myAnimal.makeSound(); // Calls Dog's implementation
    }
}

Conclusion

The abstract keyword in Java is a fundamental tool for defining abstract classes and abstract methods, promoting code organization and enforcing method implementations in subclasses. Abstract classes serve as a blueprint for concrete subclasses, ensuring consistency in behavior while allowing flexibility in implementation.
