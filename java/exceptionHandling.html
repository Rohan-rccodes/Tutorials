Exception

Exceptions are the runtime errors. it occurs depending on the data entered by the user or either at the time of type conversion. Exceptions are the errors which are not handled by the compiler. If we don't handle the exceptions, then JVM will handle it and the program will terminate abnormally

Exception Handling

Exception handling in Java is a mechanism that allows you to gracefully handle unexpected or exceptional situations in your code. These exceptions can occur during runtime and may disrupt the normal flow of your program.

In java, there are multiple exceptions and for handling these exceptions, we are having multiple exception type classes. The topmost class in exception heirarchy is throwable. After that, we have exception class. After that, we have runtime exception class and all other exception classes comes under runtime exception.

Types of Exceptions

In Java, exceptions are categorized into two main types: Checked Exceptions and Unchecked Exceptions.

  1. Inbuilt Exceptions :
    • Checked Exceptions: These are exceptions that are checked at compile time. They must be either caught (handled) using try-catch blocks or declared to be thrown using the throws keyword in method signatures. Examples include IOException and SQLException.
    • Unchecked Exceptions (Runtime Exceptions): These exceptions are not checked at compile time and can occur at runtime. They do not require explicit handling or declaration. Examples include NullPointerException and ArithmeticException.
  2. User defined Exceptions :
    • Checked Exceptions : User defined exceptions are checked exceptions only.

Key Blocks, Functions, and Keywords

To handle the exception in java, we will use the following keywords and blocks.

  • Try block : The try block is used when the programmer scans some instructinos which can occure exception. Try cannot be used alone. It alwaus be used either with catch function or with finally block.
  • Catch function : A catch function is used with try block. Every catch function will have one argument of an exception type class. We can use multiple catch funtions with try block.
    • The order of class for catch argument should be,

      first subclass -> then super class

      because if super class is written first then it will handle the exception first.

    • Other catch functions should either be at the same level or should be the super class.
    • If a super class catch function is written before the subclass catch funtion, then it is a compilation error.
  • Throws keyword : The throws keyword is used when a function want to throw a checked exception. That exception will thrown to a function or JVM.
  • Finally block : The finally block is used to specify code that should be executed whether an exception occurs or not. It is optional but useful for cleanup operations. The finally block is written at the end or after the catch functions or we can write it with try blocl also if there is no catch function.
  • Throw keyword : The throw keyword is used to throw an exception explicitly. It is often used to throw custom exceptions.

Example of Exception Handling

Here's an example of exception handling in Java: 

public class ExceptionHandlingExample {
    public static void main(String[] args) {
        try {
            int result = divide(10, 0);
            System.out.println("Result: " + result);
        } catch (ArithmeticException e) {
            System.err.println("Error: " + e.getMessage());
        } finally {
            System.out.println("Cleanup operations here.");
        }
    }
    static int divide(int dividend, int divisor) {
        if (divisor == 0) {
            throw new ArithmeticException("Division by zero");
        }
        return dividend / divisor;
    }
}
